# flog_kao_kasi_ir_full 配線・書き込み・動作確認ガイド

顔(LCD1)+歌詞(LCD2)+IR同期を1台にまとめた「表示子機」用スケッチの配線と手順。

## 1. 配線表（子機Arduino ↔ LCD1 / LCD2 / IR受信）

### 共通（LCD1・LCD2ともに同じ線を並列で接続）

実部品は **SUNLIKE SC1602BSLB-XA-GB-K**（HD44780互換 16×2、反射型・黄緑LEDバックライト付き、端子14ピン＋バックライトA/K別端子）を2枚使う。ピン番号順の対応は以下。

| ピン | 名称 | 接続先 |
|------|------|--------|
| 1 | VDD | **5V（赤レール）** ←一般的な1602Aと逆！ |
| 2 | VSS | **GND（青レール）** ←一般的な1602Aと逆！ |
| 3 | VO | 半固定抵抗T103の中央足 |
| 4 | RS | Arduino D12（2枚共有） |
| 5 | R/W | GND（青レール） |
| 6 | E | LCD1→D11 / LCD2→D10（個別） |
| 7〜10 | DB0〜DB3 | 未接続 |
| 11 | DB4 | Arduino D5（共有） |
| 12 | DB5 | Arduino D4（共有） |
| 13 | DB6 | Arduino D3（共有） |
| 14 | DB7 | Arduino D2（共有） |
| A/K（基板サイドの別端子） | バックライト | A→390Ω→5V / K→GND |

> ⚠️ **SC1602B はピン1=VDD(5V)・ピン2=VSS(GND)で、市販の1602A（1=GND, 2=5V）と電源が逆。** 逆挿しは破損の恐れ。ピン番号は基板のシルク印字（1 と 14）で必ず確認する。

**コントラスト**: T103(10kΩ)の両端足を5V/GND、中央足をVO(3番)へ。VO は2枚のLCDで共有してよい。応急処置としては VO を GND 直結でも最濃コントラストで表示可能（半固定が無い/怪しいときの切り分けに使える）。

**バックライト**: SC1602B は反射型なのでバックライト無しでも明るい場所なら読める。390Ωだと控えめの明るさ（安全側）。暗い会場でもっと明るくしたい場合のみ 100Ω 程度に交換。

### 個別（E線だけLCDごとに分ける）

| 信号 | LCD1（顔） | LCD2（歌詞） |
|------|-----------|--------------|
| E    | Arduino pin11 | Arduino pin10 |

RS(12)・データ線(5,4,3,2)は2枚で共有し、E線だけを分けることで「今どちらのLCDに書くか」を切り替える構成。スケッチの `LiquidCrystal lcd1(12, 11, 5, 4, 3, 2)` / `lcd2(12, 10, 5, 4, 3, 2)` がこの配線に対応する。

### IR受信モジュール（CHQ1838D）

実部品は **CHQ1838D**（38kHz、VS1838B互換ピン配列）。受光面（レンズの凸・型番刻印面）を手前、足を下にして左から **OUT・GND・VCC**。

| モジュールピン | 接続先 |
|----------------|--------|
| OUT | Arduino pin6（D6） |
| GND | GND |
| VCC | 5V |

> ⚠️ 逆挿しは発熱・破損の恐れがあるため、通電して熱くなったら即抜く。

## 2. 電源ルール（最重要・破損防止）

- LCD2枚とIR受信モジュールの電源は、**すべて表示子機Arduino自身の 5V / GND ドメイン**から取る。
- **親機Arduinoの5Vと子機Arduinoの5Vを直結することは絶対禁止**。レギュレータへの逆流で基板が破損する。
- GND同士は任意で1本だけ接続してよい（信号の基準を揃えたい場合）。ただしIR通信は無線（赤外線）なので、**IRだけを使う本構成ではGND接続すら不要**。
- バックライトの電流でLCDが暗い/不安定なときは、A線の390Ωを介した5Vが確実に来ているか確認する。

## 3. 書き込み手順

- このスケッチは**「表示子機」専用**。音楽子機（hack-koP2）とは**別のArduino**に焼くこと（同じ基板に両方は載らない）。
- 手順:
  1. Arduino IDE で `flog_kao_kasi_ir_full/` フォルダ（このスケッチ）を開く。
  2. ライブラリマネージャで **IRremote 4.x** が入っていることを確認（`#include <IRremote.hpp>` は4.x系のAPI）。
  3. 使用ボードを選択し、書き込み。
- `childId` は既定で 0（先頭の声部）。別の声部に割り当てたい場合のみ冒頭の `int childId` を変更して焼き直す。

## 4. 動作確認手順

1. **書き込み直後**: LCD1に静止したカエルの顔、LCD2に `MATTETA KAERU` / `OSHITE START!` の待機メッセージが出ることを確認。
   → 何も出ない/薄い場合は配線か、コントラスト用可変抵抗（V0）を回して調整する。
2. **親機（`hack_oya_kai2_2`）を起動**して参加させる: シリアルで `s`（開始）→ `0`（childId 0を参加）を送る。物理ボタン運用なら「開始ボタン」→「D4ボタン」。BPM は親機の A5 ポテンショメータで 60〜150 を連続調整できる。
3. **成功状態**: tickごとにLCD2の歌詞が1つずつ進み、LCD1の顔が口パクし、2tickで1マス右へ移動していけば成功。
4. **反応しない場合**: シリアルモニタ（**115200 baud**）を開き、NEC受信ログ（`printIRResultShort` 出力）が出ているか確認する。
   - ログが**出ない** → IRの配線（OUT→pin6）か、IR受信モジュールの向き（受光面が親機側を向いているか）を疑う。
   - ログは**出るのに動かない** → 受信 `command`（mask）の **bit0** が立っているか確認（`childId=0` は bit0）。親機側で自分の声部が参加登録されているかを見直す。

## 5. トラブルシュート

| 症状 | 最優先で確認する箇所 |
|------|----------------------|
| 片方のLCDだけ映らない | 映らない側のE線（LCD1=pin11 / LCD2=pin10）の接触。E線がLCDごとに独立しているのが本構成の要。 |
| 両方のLCDが映らない | ① V0コントラスト（可変抵抗）→ ② RS(pin12)→ ③ 共有データ線(5,4,3,2) の順で接触を確認。 |
| 文字化けする | 共有データ線(5,4,3,2)の接触不良。1本浮くと全LCDが化ける。 |
| 顔は動くが歌詞が進まない/その逆 | E線の取り違え（11と10の入れ替わり）。lcd1=11・lcd2=10 を再確認。 |
| バックライトすら点かない | バックライト回路（A→390Ω→5V / K→GND）を確認。ただし **SC1602B は反射型のためバックライトが暗くても表示自体は可能**。バックライト不点灯と表示不能は切り分けて考える。 |

## 6. 親機と同期方式（重要）

- 親機は **`hack_oya_kai2_2`** を使用する（1行目にパス文字列が混入していたビルドエラーは修正済み）。この親機は IR の `address` に tickCount 下位8bit（`tickLow`）を乗せて送るため、表示子機は歌詞位置を絶対復元でき、IRを取りこぼしても次の受信で正しい位置へ自己修復する。
- 親機の **BPM は A5 のポテンショメータで 60〜150** を連続調整する。
- **親機を reset→start し直したとき**は、表示子機の Arduino 基板の **リセットボタン（RESET）を押して `joinBase` を初期化**すること（このスケッチにソフトのリセットは無く、基準tickの初期化は基板リセットで行う）。押さないと前回の基準tickが残り、歌詞位置がずれる。音楽子機（`hack-koP2`）も同様に `localPos` が残るため、**やり直すときは全子機をリセット**するのが確実。
- **旧親機 `hack_oya_kai2`（無印・`address=0x00`固定）を使う場合**は、表示子機スケッチ冒頭の `#define USE_TICKLOW_SYNC` を **0** に変えて焼き直す（単純インクリメント方式に戻る）。
